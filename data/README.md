# 공고별 기준 데이터 큐레이션

## 왜 필요한가

청약홈 API(`api/proxy.js`)는 공고 목록만 반환합니다. **실제 소득·자산 기준은 각 공고 PDF 안에** 있고, 공고마다 달라요.

예:
- 신혼부부 특별공급에서 어떤 공고는 맞벌이 160%, 어떤 공고는 140%만 허용
- 자산 기준(부동산·자동차)도 공고별로 상이
- 일반공급에는 소득 기준이 없는 공고도 있음

→ 프론트에서 "이 공고는 160% 기준이야"라고 가정하면 사용자가 오해합니다. 그래서 **공고 PDF를 읽고 실제 기준을 `criteria.json`에 수동 입력**합니다.

## 워크플로우 (사용자 ↔ Claude)

### Step 1. 사용자가 분석 요청

관심 있는 공고를 발견하면 Claude에게 요청:

```
"반포 래미안 트리니원 공고 분석해서 criteria.json에 추가해줘"
+ PDF 파일 첨부 또는 공고문 URL
```

### Step 2. Claude가 PDF 분석

Claude는 PDF에서 다음 정보를 추출합니다:

1. **주택관리번호** (예: `2024000101`) — 공고 최상단에 표기
2. **소득 기준** — "입주자모집공고 자격 요건" 섹션
   - 신혼부부 특별공급: 외벌이 / 맞벌이 % 기준
   - 생애최초 특별공급: 외벌이 / 맞벌이 % 기준
   - 일반공급: 소득 기준 적용 여부
   - 신생아 특례: 해당 공고 적용 여부
3. **자산 기준** — "자산 보유 기준" 섹션
   - 부동산 보유액 상한 (만원)
   - 자동차 기준가액 상한 (만원)
   - 금융자산 상한 (해당 시)
4. **공고문 URL** — 원본 PDF 주소
5. **특이사항** — 특별공급 한정, 세대수 제한 등

### Step 3. `criteria.json`에 append

`announcements` 객체에 주택관리번호를 키로 한 엔트리를 추가합니다:

```json
{
  "announcements": {
    "2024000101": {
      "name": "반포 래미안 트리니원",
      "incomeStandards": {
        "신혼부부_외벌이": 140,
        "신혼부부_맞벌이": 160,
        "생애최초_외벌이": 130,
        "생애최초_맞벌이": 160,
        "일반공급": 100,
        "신생아_특례": 200
      },
      "assetStandards": {
        "부동산_만원": 21500,
        "자동차_만원": 3708,
        "금융자산_만원": null
      },
      "sourceUrl": "https://.../공고문.pdf",
      "analyzedAt": "2026-04-15",
      "notes": "특별공급 한정. 일반공급은 소득 기준 미적용."
    }
  }
}
```

### Step 4. 커밋 & 배포

```
git add data/criteria.json
git commit -m "Add criteria for 반포 래미안 트리니원"
git push
```

Vercel이 자동 재배포 → 해당 공고 카드가 **"✓ 분석됨"** 뱃지로 전환되고 실제 기준이 노출됩니다.

## UI 표시 규칙

| 상태 | 공고 카드 표시 |
|------|---------------|
| `criteria.json`에 엔트리 있음 | `✓ 분석됨` 뱃지 + 실제 소득/자산 기준 |
| `criteria.json`에 엔트리 없음 | `~ 일반 기준 추정` 뱃지 + 160% 기준 임시 표시 |

## 필드 설명

### `incomeStandards` (소득 기준)
값은 **도시근로자 월평균소득 대비 %**입니다.

- `신혼부부_외벌이`: 배우자 1명만 소득이 있는 신혼부부 특공 기준
- `신혼부부_맞벌이`: 양쪽 다 소득이 있는 신혼부부 특공 기준
- `생애최초_외벌이` / `생애최초_맞벌이`: 생애최초 특공 기준
- `일반공급`: 공공분양 일반공급 기준 (민간분양은 보통 소득 기준 없음 → `null`)
- `신생아_특례`: 신생아 특공 기준

적용 안 되는 기준은 `null`로 두세요.

### `assetStandards` (자산 기준)
단위: **만원**. 적용 안 되는 경우 `null`.

- `부동산_만원`: 세대 구성원 보유 부동산 합계 상한
- `자동차_만원`: 자동차 기준가액 (최고가 기준)
- `금융자산_만원`: 예금·주식 등 (드물게 적용)

## 주의사항

- **주택관리번호**는 API 응답의 `item.주택관리번호`와 정확히 일치해야 매칭됩니다.
- 같은 공고라도 분양 구분(민간/공공)에 따라 기준이 다르니, 공고문의 "자격 요건" 섹션을 꼼꼼히 확인하세요.
- 추정값을 넣지 말고, 명시되지 않은 기준은 `null`로 두는 편이 정확합니다.
- `analyzedAt` 은 실제 분석한 날짜를 YYYY-MM-DD로 기록합니다.
